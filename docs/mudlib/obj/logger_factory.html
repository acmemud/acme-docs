    The LoggerFactory. This service object is responsible for instantiating new logger objects and configuring them from properties files located throughout the filesystem. When new loggers are created, they are added to a pool so that they may be reused again in a later execution without the overhead of re-configuration. The factory is responsible for cleaning up logger objects which are no longer in use.

      • get_logger

        public varargs object get_logger(mixed category, object rel, int reconfig)
        Retrieve a logger instance for the given category from the pool, or create a new one from configuration. A category is represented as a hierarchical string of the form, "supercat.category.subcat.<...>". The category may also be specified as a filesystem path, in which case the path delimiters (forward slashes) will be converted to the category delimiter (periods).
        Parameters:
        category - an object or string representing the category; if an object is specfied, it's program_name(E) will be used.
        rel - optional object to use for resolving relative paths in configuration files; if unspecified, previous_object() will be used.
        reconfig - set this flag to 1 if you need to re-read the logger configuration
        Returns:
        a logger instance

      • parse_output_prop

        protected mixed *parse_output_prop(string val)
        Translate the property value of an output property to something more structured than a string. The result will be of the form: ({ ({ int type, string target }), ... }) where type is one of 'c' or 'f' and target is an object spec or a file path, for console output or file output, respectively.
        Parameters:

        Returns:

        public int release_logger(mixed category, string euid)
        Releases a logger object from the logger pool, thereby removing any references to it held by the factory. If there are no other references, the logger will also be destructed.
        Parameters:
        category - an object or string representing the category; if an object is specfied, it's program_name(E) will be used.
        euid - the euid of the logger to release
        Returns:
        1 if a logger was released, 0 if no logger was found
