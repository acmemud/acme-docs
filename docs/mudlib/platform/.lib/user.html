      • attach_session

        public string attach_session(object interactive, string user_id)
        Attach a user to their user session. If it can be safely resumed, their last session will be used. Otherwise, a new session will be created with a new avatar. Then the "descension" process takes place, which involves resuming their session and switching their connection from the provided interactive (which will generally be the login object).
        Parameters:
        interactive - the user's current interactive
        user_id - the user id to attach to session
        Returns:
        the attached session id
