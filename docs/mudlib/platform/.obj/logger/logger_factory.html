    The LoggerFactory. This service object is responsible for instantiating new logger objects and configuring them from properties files located throughout the filesystem. When new loggers are created, they are added to a pool so that they may be reused again in a later execution without the overhead of re-configuration. The factory is responsible for cleaning up logger objects which are no longer in use.

      • get_logger

        public varargs object get_logger(mixed zone, object rel, int reconfig)
        Retrieve a logger instance for the given zone from the pool, or create a new one from configuration. A zone is represented as a hierarchical string of the form, "superzone.zone.subzone.<...>". The zone may also be specified as a filesystem path, in which case the path delimiters (forward slashes) will be converted to the zone delimiter (periods).
        Parameters:
        zone - an object or string representing the zone; if an object is specfied, it's load_name(E) will be used.
        rel - optional object to use for resolving relative paths in configuration files; if unspecified, previous_object() will be used.
        reconfig - set this flag to 1 if you need to re-read the logger configuration
        Returns:
        a logger instance

        Read in logger configuration for the specified zone. Starting in the specified directory, this function will look for the file "etc/logger.properties", and inspect the file for any configuration properties which apply to the specified zone. The result is a mapping of the form: ([ "output" : ({ ({ int type, string target }), ... }), "format" : str format_string, "level" : str level ]) FUTURE better to use a struct than a mapping here

      • release_logger

        public int release_logger(mixed zone, string euid)
        Releases a logger object from the logger pool, thereby removing any references to it held by the factory. If there are no other references, the logger will also be destructed.
        Parameters:
        zone - an object or string representing the zone; if an object is specfied, it's load_name(E) will be used.
        euid - the euid of the logger to release
        Returns:
        1 if a logger was released, 0 if no logger was found

      • clean_up_loggers

        protected int clean_up_loggers()
        Clean up stale loggers, called once per reset. A logger is considered stale if it hasn't been referenced in 5 minutes, and is referenced by nothing except the LoggerFactory.
        Returns:
        the number of logging zones released (may be more than the number loggers destructed if loggers are shared between zones)
